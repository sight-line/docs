    Introduction

    API endpoint

    https://api.sight-line.uk

    Examples in this documentation are written using httpie.

    To install httpie on OS X run brew install httpie

    Authentication

    The Sight Line API uses Token Authentication.

    Acquiring a Authentication Token

    Authentication Tokens are granted on request.

    Please visit here to request a token.

    You will need to register the following details:

    • Email
    • Organisation
    • Address
    • Telephone

    Authenticating Requests

    $ http "https://api.sight-line.uk/sites/" \
    "Authorization: Token $auth_token"

    {
    "uuid": "b07e9578-24e5-47de-8c56-ed82508f9ed2",
    "first_sign": {
        "uuid": "1cbf7316-6ed3-4cbe-a9d7-27a7c9b04e66",
        "owner_uuid": "60fe3539-056a-4333-912f-cae8ed206388",
        "owner_email": "streetworks@streetworks.info",
        "device_id": 1,
        "major_id": 2,
        "minor_id": 1,
        "latitude": 51.5349945,
        "longitude": -0.0552559,
        "sign_type": "RIGHT"
    },
    ...
    "active": true,
    "source": "streetworks"
}

    All requests must be authenticated with an access token supplied in the Authorization header using the Token scheme.

    Clients may only have one active access token at a time, per user.

    Acquiring a new access token will invalidate any other token you own for that user.

    $ http --form POST "https://api.sight-line.uk/sites/sign/" \
    "Authorization: Token $auth_token" \
    "device_id=$device_id" \
    "major_id=$major_id" \
    "minor_id=$minor_id" \
    "latitude=$latitude" \
    "longitude=$longitude" \
    "sign_type=$sign_type"

    Creates a specified Sign.

    The combination of Device, Major and Minor ID’s must be unique otherwise they will replace an existing Sign.

    Request arguments
    Parameter Description
    device_id
    Required    		 Device ID (Integer)
    major_id
    Required    		 Major ID (Integer)
    minor_id
    Required    		 Minor ID (Integer)
    latitude
    Required    		 Latitude (Float)
    longitude
    Required    		 Longitude (Float)
    sign_type
    Required    		 Sign Type of the sign. See Sign Type choices
    Response arguments
    Parameter Description
    uuid Sign’s uuid
    owner_uuid Sign owner’s uuid
    owner_email Sign owner’s email
    device_id Device ID (Integer)
    major_id Major ID (Integer)
    minor_id Minor ID (Integer)
    latitude Latitude (Float)
    longitude Longitude (Float)
    sign_type Sign Type of the sign. See Sign Type choices

    "minor_id=$minor_id" \
    "latitude=$latitude" \

    Report

    Errors with specific Sites can be reported.

    Authentication is NOT required for this API.

    Report a specific site

    $ http POST "https://api.sight-line.uk/sites/report/" \
    "Authorization: Token $auth_token" \
    "site==$site_uuid"

    {
  "uuid": "91d3536e-bb5b-4b8b-9dec-4fae3a35528e",
  "site": "cd54227c-664c-4631-bc19-d42a0440b5f7"
}

    Flags a specific Site as having erroneous data.

    Request arguments
    Parameter Description
    site
    Required    		 The uuid of the Site.
    Response arguments
    Parameter Description
    uuid The report’s uuid.
    site The specified site’s uuid.
